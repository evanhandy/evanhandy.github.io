The <video> Element

The <video> element was added in HTML5 and has great support on all modern browsers. Before the <video> element was created videos could only be viewed by using plug-ins, like flash.

If a browser does not support the <video> element, you can choose to have some text display instead. For example, in a not supporting browser this code would display "Your browser does not support the video tag."

    <video width="600" controls>
        <source src="2015-10-110-god-will-help-you-veme-360p-eng.mp4" type="video/mp4">
        Your browser does not support the video tag.
    </video>

<video> Attributes

The <video> element has a few attributes:

  • width
  • height
  • autoplay
  • controls

The width and height attributes are used to set the width and height of the video. Setting the width is recommended to help prevent potential flickering as the page loads. The autoplay attribute has no value and causes the video to automatically play when the page loads. We'll go into the controls attribute in the next section.

The controls Attribute

The controls attribute is crucial for usability. Like autoplay, it doesn't have any value to set, it just needs to be present inside the <video> tag. If you don't put it in your <video> tag, the video will not be easily controlled by the user, potentially causing frustration with your site. While you may not need it in every situation, leaving it out should be a concious choice, not a mistake.

Below are some examples of two <video> elements, one with the controls attribute, and one without.

<video> Element With controls

    <video width="600" controls>
        <source src="2015-10-110-god-will-help-you-veme-360p-eng.mp4" type="video/mp4">
        Your browser does not support the video tag.
    </video>

<video> Element Without controls

    <video width="600">
        <source src="2015-10-110-god-will-help-you-veme-360p-eng.mp4" type="video/mp4">
        Your browser does not support the video tag.
    </video>

The <source> Element

The <source> is a void element used to get the video content. It is nested inside the <video> element.

    <video width="600" controls>
        <source src="2015-10-110-god-will-help-you-veme-360p-eng.mp4" type="video/mp4">
        Your browser does not support the video tag.
    </video>

The <source> element has two attributes:

  • src, very straightforward, treated exactly like the src attribute in the img element.
  • type, describes what type of file (e.g. type="video/mp4")

Video File Types

There are three file formats supported for the <video> element, each with a corresponding media type:

  • MP4 = video/mp4
  • WebM = video/webm
  • Ogg = video/ogg

There is mixed support of these across all browsers, with MP4 being the most supported.
